# 水电表管理功能 - 产品需求文档（PRD）

## 📋 功能概述

开发一个水电表管理模块，用于记录和管理合租房的水电表读数，计算费用，并生成微信消息通知租户。

## 🎯 核心需求

### 1. 水电表录入
**用户故事**：作为房东，我需要快速录入每个房间的水电表读数，以便计算水电费用。

**功能点**：
- 选择房间
- 自动显示该房间的上次水表、电表读数
- 输入本次水表、电表读数
- 自动计算用量和费用
- 支持批量录入多个房间
- 保存记录到数据库

### 2. 水电记录查询
**用户故事**：作为房东，我需要查看历史水电记录，以便核对费用。

**功能点**：
- 按房间筛选
- 按日期范围筛选
- 按类型筛选（水/电）
- 显示完整历史记录
- 支持导出Excel

### 3. 微信消息生成
**用户故事**：作为房东，我需要生成水电费用通知，方便通过微信发送给租户。

**功能点**：
- 选择一条水电记录
- 自动生成格式化的微信消息
- 一键复制消息内容
- 支持自定义消息模板

## 📊 数据模型

### 现有模型（已存在）
```python
class UtilityReading(Base):
    id: int
    room_id: int              # 房间ID
    utility_type: str         # 类型：water/electricity
    reading: Decimal          # 本次读数
    reading_date: Date        # 抄表日期
    previous_reading: Decimal # 上次读数
    usage: Decimal            # 用量（自动计算）
    amount: Decimal           # 金额（自动计算）
    rate_used: Decimal        # 使用的费率
    recorded_by: int          # 记录人
    notes: Text              # 备注
    created_at: DateTime
    updated_at: DateTime
```

### 需要扩展的功能
1. **批量录入**：一次性录入一个房间的水和电
2. **微信消息模板**：可配置的消息格式

### 交互流程
1. **录入流程**
   ```
   选择房间 → 自动加载上次读数 → 输入本次读数
   → 实时计算用量和费用 → 确认保存 → 显示成功
   ```

2. **生成微信消息流程**
   ```
   查看记录列表 → 点击"生成微信消息"按钮
   → 弹窗显示格式化消息 → 点击"复制"
   → 打开微信粘贴发送
   ```

## 🔧 技术实现

### 后端 API（已存在，需确认）
- `POST /api/v1/utility-readings` - 创建水电记录
- `GET /api/v1/utility-readings` - 查询列表
- `GET /api/v1/utility-readings/{id}` - 查询详情
- `GET /api/v1/rooms/{room_id}/utility-readings` - 房间的水电记录

### 需要新增的 API
- `POST /api/v1/utility-readings/batch` - 批量录入（水和电同时）
- `GET /api/v1/utility-readings/wechat-message/{id}` - 生成微信消息

### 前端页面
- `UtilityView.vue` - 重构，增加录入和微信消息功能
- `UtilityReadingForm.vue` - 录入表单组件
- `WechatMessageDialog.vue` - 微信消息弹窗组件

## ⚡ 性能要求

- 录入响应时间 < 500ms
- 列表加载时间 < 1s
- 微信消息生成 < 200ms

## 📝 注意事项

1. **数据一致性**：水和电的记录应该分开存储，但录入时可以一起
2. **费率管理**：确保使用正确的费率（最新生效的费率）
3. **用户体验**：实时反馈，自动计算，减少用户输入
4. **错误处理**：清晰的错误提示，帮助用户快速定位问题
